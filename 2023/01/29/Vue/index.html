初识Vue

  1. 想让Vue工作，需要创建一个Vue实例，且要传入一个配置对象
  2. 容器内的代码必须符合html规范，混入一些特殊的Vue语法
  3. 容器内的代码被称为Vue模板
  4. 一个容器对应一个Vue实例，反之一样，真实开发中只有一个Vue实例，配合组件一起使用
  5. 中的xxx要写js表达式，且能读到data中的所有属性
  6. 一旦data内数据发生改变，那么模板中用到该数据的地方也会自动更新
  7. 编程范式分为命令式编程和函数式编程，jq为命令式，一步一步指定需要做什么，vue为声明式，可以做到数据和页面分离，声明一个事件，告诉vue怎么做就可以了，复杂的dom操作无需亲自来

MVVM模型

vue在一定程度参考了此模型

  1. M：模型（Model）对应data中的数据
  2. V：视图（View）模板，对应Dom结构（vue模板间接影响了dom）
  3. VM：视图模型（ViewModel）对应Vue实例对象，日后Vue实例对象的命名，用vm

这里值得注意的是插值字符串中两个大括号能看见的东西都在Vue实例化对象中，data内的东西也会给数据代理到Vue实例化对像中去，成为其中的一个属性，且Vue实例化对象中原型对象上的所有属性，在Vue模板中都可以直接使用。

数据代理

Object.defineProperty()

在vue2中很多功能实现都靠这个函数，他就像一座桥梁，把两个无关系的值捆绑到一起，一者变另一者也变。

  1. Vue中的数据代理：通过vm对象来代理data对象属性中的操作（读/写）
  2. 基本原理：
    • 通过Object.defineProperty()把data对象所有的属性添加到vm上
    • 为每个添加到vm上的属性都指定一个getter/setter
    • 在getter/setter内部去操作（读/写）data中对应的属性

事件处理

事件的基本使用：

  1. 使用v-on或者简写@在标签中绑定对象， 如@click
  2. 事件的回调需要配置在methods对象中，最终还是在vm上
  3. methods中配置的函数，不要用箭头函数，这会将原本指向vm的this改变
  4. methods配置的函数都是被Vue管理的函数，this的指向是vm
  5. @click='demo'和@click ='demo($event)'效果一致，但是后者可以传参，demo为函数

事件修饰符

  1. prevent：阻止默认事件（常用）
  2. stop：阻止事件冒泡（常用，默认处理事件的为事件冒泡而不是事件捕获）
  3. once：事件只触发一次（常用）
  4. capture：使用事件的捕获方式；
  5. self：只有event.target是当前操作的元素时才触发事件
  6. passive：事件的默认行为立即执行，无需等待事件回调执行完毕
  7. 注意：事件修饰符是可以连着起写的，如：@click.stop.once

键盘事件

  1. Vue中常用的键盘别名
    • 回车：enter
    • 删除：delete
    • 退出：esc
    • 空格：space
    • 换行：tab
    • 上：up
    • 下：down
    • 左：left
    • 右：right
  2. Vue中未提供别名的按键，可以使用按键原始的key去绑定，但注意要转为kebab-case（短横线命名）
  3. 系统修饰键（用法特殊）：ctrl、alt、shift、mete
    • 配合keyup使用：按下修饰键的同时，再按下其他键，随后释放其他键，事件才会被触发
    • 配合keydown使用：正常触发事件
  4. 可以使用keycode指定具体的按键（已经弃用）
  5. 可以自定义键名：Vue.config.keyCode.自定义键名=键码（不推荐）

计算属性

  1. 定义： 要用的属性不存在，要通过已有属性计算得来
  2. 原理：底层借助了Object.defineproperty方法提供的getter和setter。
  3. get函数在初次读取时会执行一次，当依赖的数据发生改变时会再次被调用
  4. 优势：与methods实现相比，内部有缓存机制（复用），效率更高，调试也方便，因为计算属性会显示到vue开发者工具，而方法的内容不会
  5. 备注：计算属性最终会出现在vm上，直接读取使用即可

监视属性

监视属性watch：

  1. 当监视属性变化时，回调函数自动调用，进行相关操作
  2. 监视的属性必须存在，才能进行监视
  3. 监视的两者写法：
    • new Vue传入
    • vm.$watch监视

以上两种写法，如果事先知道需要监视谁，那就写写法一，如果事先不知道，那就写写法二。

深度监视

  1. Vue种的watch默认不监测对象内部值的改变（只监视一层），但是Vue本身是可以监测多层数据的变动的，只是在watch默认关闭
  2. 配置deep: true可以开启检测对象内部值的改变，默认为false
  3. 当简单类型数据（只有一层时）就不要开深度监视了，提高效率

深度监视的简写

当配置项内只有handler属性时，可以简写，简写后deep等属性不能进行配置。

计算属性computed和监视属性watch的区别

  1. 前者能完成的功能后者也能完成
  2. 但是后者能完成的功能前者不一定能完成，比如watch可以进行异步任务而computed不可以
  3. 两个重要小原则：
    • 所有被Vue管理的函数，最好写成普通函数，this为vm，或者组件实例对象
    • 所有不被Vue管理的函数（定时器回调函数，ajax回调函数等）最好写成箭头函数，这样this指向才是vm或者组件实例化对象

key的原理

以上代码的写法不会出现数据错乱的问题，因为key = 一个id

key的作用：

  1. 虚拟dom的中key的作用：

    key是虚拟dom中的标识，当数据发生改变时，Vue会根据新数据生成新的虚拟dom，随后Vue进行新虚拟dom的差异比较

  2. 对比规则：

    • 旧虚拟dom中找到与新虚拟dom相同的key，若虚拟dom中的内容没变，直接使用之前的真实dom；若虚拟dom的内容变了，则生成新的真实dom，随后替换屌页面中之前的真实dom
    • 若旧虚拟dom中未找到与新虚拟dom中相同的key，就会创建新的真实dom随后渲染到页面

  3. 用index作为key可能会引发的问题：

    • 若对数据进行逆序添加、逆序删除等破坏顺序的操作会产生没有必要的真实dom更新，达不到组件复用的效果，造成效率低
    • 如果结构中含有输入类的dom则会产生错误的dom更新，因为输入类dom的操作数据是在真实dom上残留的，虚拟dom内无数据，那Vue就会以为key一致的dom内的输入节点dom就是一样的，就会重复使用，导致页面数据错乱

  4. 如何选择key？

    • 最好使用每条数据唯一的表示作为key，比如id、手机号、身份证、学号等
    • 如果不存在对数据的逆序添加删除等破坏顺序的操作，仅用于渲染列表表示展示，使用index作为key的值是没有问题的

Vue监测数据的原理

监视数据的原理：

  1. Vue会监视data的所有层次的数据

  2. 如何监测对象中的数据？

    通过setter实现监视，且要在new Vue时就传入要监测的数据。

    • 对象中追加的属性，Vue默认不做响应式处理

    • 如需给后添加的属性做响应式，请使用如下API：

      Vue.set(target, propertyName/index, value)

      vm.$set(target, propertyName/index, value)

  3. 如何监测数组中的数据？

    通过包裹数组更新元素的方法实现，本质做了两件事：

    • 调用原生对应的方法对数组进行更新
    • 从新解析模板（重写那七个方法），进而更新页面

  4. 在Vue修改数组中的某个元素一定要用如下方法：

    • api：push()、pop()、shift()、unshift()、splice()、sort()、reverse()
    • Vue.set()、vm.$set()

    特别注意：Vue.set()和相关方法不能给vm和vm的跟数据对象添加属性

数据劫持会将从data中获得的数据获得一个响应，从而触发解析模板的响应

收集表单数据

若：<input type='text'/>，则v-model收集的是value值，用户输入的就是value值。

若： <input type='readio'>，则v-model收集的是value值，且要给标签配置value值。

若：<input type='checkbox'>没配置value，则v-model收集的是checked值（布尔类型），若配置：

  • v-model初始值为非数组：收集的是checked
  • v-model初始值为数组：收集的是value组成的数组

v-model的三个修饰符：

  • lazy：失去焦点再收集数据
  • number：输入字符串再转为有效数字
  • trim：输入首尾空格过滤

自定义指令

定义语法：

  1. 局部指令：directives: {指令名: 配置对象}或者directives: {指令名: 回调函数}
  2. 全局指令：Vue.directive(指令名, 配置对象)或者Vue.directive(指令名, 回调函数)

配置对象中常用的三个回调：

  1. bind：指令与元素成功绑定时调用
  2. inserted：指令所在元素被插入页面时调用
  3. update：指令所在模板结构被重新解析时调用

备注：

  1. 指令定义时不加v-，使用时需要v-
  2. 指令名如果是多个单词，要使用user-name命名方式，不要用这种：userName

生命周期

常用的生命周期钩子：

  1. mounted：发送ajax请求、启动定时器、绑定自定义事件、订阅消息等初始化操作
  2. beforeDestroy：清除定时器、解绑自定义事件、取消订阅消息等收尾工作

关于销毁Vue实例

  1. 销毁后借助Vue开发者工具看不到任何信息
  2. 销毁后自定义事件会失效，但是原生的DOM事件亦然有效
  3. 一般不会在beforeDestroy操作数据，即使操作数据也不会触发更新流程

父子组件的通信

在开发中，往往有一些数据需要从上层传递到下层：比如一个页面中，我们从服务器中请求到了很多数据，其中一部分数据并非是我们整个页面的大组件来显示的，而是需要下面的子组件进行展示，这个时候并不会让子组件再发送一次网络请求，而是直接让父组件将数据传递给小组件。

有两种方法可以实现需求：

  • 通过props向子组件传递数据（父传子）
  • 通过时间向父组件发送消息（子传父）

ref属性

  • 被用来给元素或者子组件注册引用信息（id的替代者）
  • 应用再html标签上获取的是真实的DOM元素，应用在组件标签上则是组件实例对象（VC）
  • 使用方式：打标识：<h1 ref='xxx'>...</h1>，获取：this.$refs.xxx

配置项props

让组件接收外部传进来的数据

  1. 传递数据：<demo name='xxx'/>
  2. 接收数据：
    • 只接收：props:['name']
    • 限制类型：props{name: Number}
    • 限制类型、限制必要性、指定默认值：props:{name: {type: String, required: true, default: 'xx'}}

值得注意的是，prop属性是只读的，不能进行修改

，若要修改需要复制一份到data，更改data内的即可

路由

  • 路由是通过互联的网络把信息从源地址传输到目的地址的活动
  • 路由器提供两种机制：路由和转送，路由决定了数据包从来源到目的地的路径；转送将输入端的数据转移到合适的输出端，路由内有映射表，映射表会将内网ip绑定电脑mac地址，这样数据可以先查找公网ip进入路由，再由路由通过映射表转发到对应内网ip所代表的输出端

后端渲染和后端路由：后端渲染指的是在服务器利用如jsp等技术在服务器渲染出一个网页（html+css）然后发给前端直接展示的一个流程，也称之为服务端渲染；后端路由指的是后端处理url和页面之间的映射关系

前端渲染和前端路由：前端渲染是指浏览器中宣誓的大部内容都是由前端编写的js代码在浏览器中执行最终渲染出来的网页；前端路由指的是url和页面之间的映射关系由前端进行管理，后端不再管理

修改网页/后缀而不刷新的方式：

  • location.hash = 'aaa'，改变哈希值实现
  • history.pushState({}, '', 'home')，入栈实现
  • history.back()，返回上一级
  • history.replaceState({}, '', 'about')，没有历史记录的实现
  • history.go(-1)，返回上一级，这个等价于history.back()，当然也可以任意正数负数
  • history.forward()，前进一级
